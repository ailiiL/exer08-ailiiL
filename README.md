## APP DESCRIPTION 
This application is a simple Expense Tracker that allows users to record and manage their expenses. Data is stored using Firebase, so entries persist even after the app is closed. Users can add expenses by providing a name, description, category, amount, and payment status. All expenses are displayed on the main page, where users can view, edit, or delete entries. The app also includes proper form validation to ensure correct input.

## THINGS I DID IN THE CODE
I implemented CRUD operations using Firebase Firestore and displayed data in real-time using StreamBuilder. I used the Provider package for state management to handle adding, editing, deleting, and updating expense status. The project follows a clean structure by separating models, providers, and UI. I also added form validation to prevent invalid inputs.
